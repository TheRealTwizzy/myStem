# End-to-End UX Flows

## Flow 1 — First launch and first Stemling

1. Open app.
2. Accept required terms and privacy disclosures.
3. Enter Hollow setup.
4. Select biome family.
5. Select sub-biome.
6. Review default conditions.
7. Create New Seed.
8. View mandatory first Stemling tutorial.
9. Meet fixed-form Stemling.
10. Perform first observation.
11. Make first Hollow adjustment.
12. Receive immediate biofeedback.
13. Exit with clear return expectation.

Success criteria:

- no subscription required;
- no aggressive paywall;
- fixed anatomy correct;
- first action understandable;
- tutorial can be disabled afterward.

## Flow 2 — Routine care

1. Open Hollow.
2. View current biological condition.
3. Inspect observations.
4. Adjust one or more controls.
5. Server confirms change.
6. Stem responds.
7. Session ends or continues.

Offline:

- controls disabled;
- cached view available;
- no queued changes.

## Flow 3 — Becoming and Consent

1. Becoming readiness appears through multimodal signals.
2. Player opens observation.
3. System describes broad process.
4. Player chooses Stabilize, Suppress, or Let Continue.
5. Intervention flow runs if applicable.
6. Server resolves anatomy atomically.
7. Stage and sustainment update.
8. One Field Entry is created.
9. Player reviews the record.

## Flow 4 — Stabilize

1. Confirm Stabilize.
2. View supportive condition guidance.
3. Maintain conditions online.
4. If interrupted before commitment, offer one Resume Support opportunity.
5. Resume or allow resolution.
6. Commit result.
7. Create Field Entry.

## Flow 5 — Suppress

1. Confirm Suppress.
2. View opposing condition guidance.
3. Maintain uninterrupted attempt.
4. If broken, suppression opportunity ends for that Becoming.
5. Resolve with possible redirection or Mutation Debt.
6. Create Field Entry.

## Flow 6 — Let Continue

1. Confirm Let Continue.
2. Explain that natural variability remains broad.
3. Resolve from current history and biology.
4. Create Field Entry.

## Flow 7 — Premium illustration

1. Open eligible Free Field Entry.
2. View premium explanation and no-reroll rule.
3. Validate paid entitlement and allowance.
4. Confirm generation.
5. Reserve allowance and lock card.
6. Generate and validate.
7. On success, attach image and archive.
8. On failure, restore allowance and leave card unchanged.

## Flow 8 — Lineage Synthesis

1. Open eligible Premium Field Entry.
2. Select Synthesize.
3. Review permanent Genetic Entry consumption.
4. Select empty Hollow.
5. Select biome family and sub-biome.
6. Final confirmation.
7. Atomic synthesis.
8. New fixed-form Stemling appears.
9. Source Genetic Entry shows Consumed.

## Flow 9 — Trade

1. List one eligible Field Entry.
2. Receive offers.
3. Compare one-for-one.
4. Select offer.
5. Revalidate ownership and locks.
6. Final side-by-side confirmation.
7. Atomic swap.
8. Close other offers.

## Flow 10 — Gift

1. Select eligible Field Entry.
2. Select recipient.
3. Send offer.
4. Recipient reviews.
5. Recipient accepts or declines.
6. Atomic transfer on acceptance.

## Flow 11 — Adult strain and recovery

1. Biofeedback indicates mismatch.
2. Player opens sustainment observation.
3. Adjusts conditions.
4. State improves or worsens.
5. If Decay persists, present direct regression warning.
6. Player attempts recovery.
7. If recovery fails while online, regression resolves visibly.
8. Post-regression recovery begins.

## Flow 12 — Release

1. Select eligible Stem.
2. View blocked-state reason if ineligible.
3. Review permanent farewell consequences.
4. Confirm.
5. Server removes living Stem atomically.
6. Hollow becomes empty.
7. Existing Field Entries remain.
8. No living history or recall appears.

## Flow 13 — Subscription lapse and grace

1. Renewal fails.
2. Platform confirms grace or expiry.
3. Capabilities update.
4. Grace preserves archive/download/synthesis.
5. Generation and grants disable.
6. Successful payment restores active state and normal grant rules.
