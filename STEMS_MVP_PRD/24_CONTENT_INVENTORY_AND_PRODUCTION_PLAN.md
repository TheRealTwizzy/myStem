## Production workstreams

### A — Component definitions

For each component:

- anatomy;
- operation;
- prerequisites;
- incompatibilities;
- environmental drivers;
- support needs;
- sustainment;
- biofeedback;
- visual prompt data;
- regression behavior;
- tests.

### B — Support components

Support content must be authored alongside Hero and Standard content, not postponed. No high-salience component is complete until its viability path exists.

### C — Observation language

Create template groups for:

- preference;
- strain;
- instability;
- decay;
- recovery;
- Becoming emergence;
- Stabilize;
- Suppress;
- Let Continue;
- The Choosing;
- regression;
- settled adult behavior.

### D — Behavior motifs

Each motif defines:

- trigger;
- anatomy dependency;
- stage;
- state;
- motion;
- sound;
- caption;
- reduced-motion equivalent.

### E — Biome content

For all 21 sub-biomes:

- description;
- five unique-variable labels;
- functional surfaces;
- threshold configuration;
- ambient visual direction;
- sound direction;
- accessibility text;
- test coverage.

### F — Premium poses

Sixteen pose families with:

- supported body plans;
- visibility requirements;
- camera range;
- scale logic;
- forbidden occlusion;
- validation examples.

## Content governance

Every content addition must pass:

- canon review;
- biological review;
- visual-safety review;
- accessibility review;
- simulation compatibility;
- AI-generation compatibility;
- fixture or automated test coverage.

## Localization

Author text with:

- short sentences;
- parameter-safe grammar;
- no embedded word order assumptions;
- glossary consistency;
- plural/count flexibility;
- no reliance on English-only puns.

## Content tooling

Required internal tools:

- component editor;
- anatomy graph viewer;
- compatibility validator;
- biome configuration editor;
- observation preview;
- pose preview;
- fixture runner;
- terminology linter;
- forbidden-content linter;
- asset provenance viewer.

## Completion definition

Launch inventory is complete only when quantity targets, validation, localization readiness, accessibility metadata, and test coverage all pass.
